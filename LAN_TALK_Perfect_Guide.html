1. Prerequisites

Before you begin, make sure you have the following installed on your system:

  • Python 3.8 or higher
  • pip (Python package manager)
  • Git (optional, for version control)
  • A modern web browser (Chrome, Firefox, Edge, etc.)
1
Check Python Version

Verify your Python installation by running the following command in your terminal:

python --version

You should see output like Python 3.8.10 or higher.

2
Check pip Installation

Verify pip is installed:

pip --version
3
Install Required System Packages

Windows users typically don't need additional packages. Just make sure you have Python installed from python.org.

Install required packages using Homebrew:

brew install python

Install required packages:

sudo apt update
sudo apt install python3 python3-pip python3-venv
Tip: Using a virtual environment is highly recommended to avoid conflicts with other Python projects.

2. Installation

1
Extract the ZIP File

Extract the LAN_TALK.zip file you received to a location on your computer.

2
Navigate to the Project Directory
cd path\to\LAN_TALK
cd path/to/LAN_TALK
3
Create a Virtual Environment
python -m venv venv
venv\Scripts\activate

You should see (venv) appear at the beginning of your command prompt.

python -m venv venv
source venv/bin/activate

You should see (venv) appear at the beginning of your command prompt.

4
Install Dependencies
pip install -r requirements.txt
Note: This may take a few minutes depending on your internet connection. The command will download and install all required Python packages.

You should see output showing the installation progress of various packages including Django, Channels, and other dependencies.

Important: Always activate the virtual environment before running any commands related to the project. You'll need to reactivate it each time you open a new terminal window.

3. Database Setup

1
Apply Database Migrations

Run the following command to set up the SQLite database:

python manage.py migrate

This command creates all necessary database tables for users, chat rooms, messages, and file sharing.

Note: You should see output showing migrations being applied for various Django apps.
2
Create an Admin Superuser

Create an administrator account to access the admin panel:

python manage.py createsuperuser

Follow the prompts to enter a username, email address, and password for the admin account.

Tip: Choose a strong password but make sure to remember it! This account will have full access to the admin panel.
3
Database Configuration (Optional)

By default, LAN_TALK uses SQLite for simplicity. If you want to use a different database like PostgreSQL or MySQL, you can modify the .env file:

# Example PostgreSQL configuration in .env
DATABASE_ENGINE=django.db.backends.postgresql
DATABASE_NAME=lantalk
DATABASE_USER=postgres
DATABASE_PASSWORD=yourpassword
DATABASE_HOST=localhost
DATABASE_PORT=5432
Warning: Using a different database requires additional setup and installing the appropriate database adapter. For most LAN deployments, the default SQLite is sufficient.

4. Running the Server

1
Start the Development Server

Run the following command to start the Django development server:

python manage.py runserver 0.0.0.0:8000
Tip: Using 0.0.0.0 as the host allows other devices on your LAN to access the application. The port 8000 can be changed if needed.

You should see output indicating that the server is running, similar to:

Starting development server at http://0.0.0.0:8000/
Quit the server with CONTROL-C.
2
Access the Application

You can now access LAN_TALK in your web browser:

  • On your computer: http://localhost:8000
  • From other devices on your LAN: http://YOUR_IP_ADDRESS:8000

Find your IP address by running:

ipconfig

Look for the IPv4 Address under your active network adapter (usually "Wireless LAN adapter Wi-Fi" or "Ethernet adapter").

Find your IP address by running:

ifconfig | grep "inet " | grep -v 127.0.0.1

Look for the inet address that's not 127.0.0.1.

Find your IP address by running:

ip addr show | grep "inet " | grep -v 127.0.0.1

Look for the inet address that's not 127.0.0.1.

3
Login or Register

When you first access the application, you'll be redirected to the login page. You can:

  • Login with the admin account you created earlier
  • Click "Register" to create a new user account
Important: The development server is not suitable for production use. For a more permanent deployment, consider using Daphne or other ASGI servers with proper security configurations.

6. Testing with Multiple Users

To fully experience LAN_TALK's real-time features, you'll want to test with multiple users. Here are some approaches:

1
Using Different Browsers

The simplest way to test with multiple users is to use different web browsers simultaneously:

  • Open Chrome and log in with one account
  • Open Firefox and log in with another account
  • Open Edge or Safari and log in with a third account
Tip: This method allows you to test chat, calls, and file sharing between different users on the same computer.
2
Using Incognito/Private Windows

Another approach is to use incognito or private browsing windows in the same browser:

Open an incognito window with Ctrl+Shift+N (Windows/Linux) or âŒ˜+Shift+N (macOS)

Open a private window with Ctrl+Shift+P (Windows/Linux) or âŒ˜+Shift+P (macOS)

Open an InPrivate window with Ctrl+Shift+N (Windows) or âŒ˜+Shift+N (macOS)

3
Using Different Devices on Your LAN

For the most realistic testing, use multiple devices on your local network:

  1. Keep the server running on your computer
  2. Connect other devices (laptops, tablets, smartphones) to the same WiFi network
  3. On each device, open a browser and navigate to http://YOUR_COMPUTER_IP:8000
  4. Log in with different accounts on each device
Note: This method is ideal for testing LAN discovery and WebRTC calls between actual devices.
4
Testing LAN Discovery

To test the LAN discovery feature, run the following command in a new terminal window (with the virtual environment activated):

python manage.py scan_lan

This command will scan your local network for other LAN_TALK users and display their status.

Important: For WebRTC calls to work properly between different devices, they must be on the same local network. Internet-based WebRTC calls would require additional TURN server configuration.

7. Troubleshooting

1
404 Error on Root URL

If you encounter a 404 error when accessing the root URL:

  • Make sure you're using the latest code with the root URL redirect
  • Check that the server is running properly
  • Try accessing /login/ directly
# Check if the root URL redirect is properly configured in lan_talk/urls.py
path('', RedirectView.as_view(url='login/', permanent=False))
2
WebSocket Connection Issues

If real-time features aren't working:

  • Ensure you're using a modern browser with WebSocket support
  • Check firewall settings if connecting from different devices
  • Verify that the Channels and Daphne packages are installed
# Verify Channels installation
pip show channels

# Check if Redis is properly configured (if using channels_redis)
pip show channels_redis
Tip: Check the browser console (F12) for any WebSocket connection errors.
3
Database Errors

If you encounter database-related errors, you can reset the database:

# Reset the database
python manage.py flush

# Re-apply migrations
python manage.py migrate

# Create a new superuser
python manage.py createsuperuser
Warning: The flush command will delete all data in the database. Use with caution!
4
WebRTC Call Problems

If voice or video calls aren't working:

  • Allow camera/microphone permissions when prompted
  • Test with different browsers if issues persist
  • Ensure both users are on the same LAN for direct connections
# Check WebRTC configuration in .env file
STUN_SERVERS=stun:stun.l.google.com:19302,stun:stun1.l.google.com:19302
Tip: WebRTC requires proper STUN servers for NAT traversal. The default configuration should work for most LAN setups.
5
Server Won't Start

If the Django server won't start:

  • Check if another process is already using port 8000
  • Verify that all dependencies are installed
  • Check for syntax errors in your code
# Check if port 8000 is in use (Windows)
netstat -ano | findstr :8000

# Check if port 8000 is in use (macOS/Linux)
lsof -i :8000

# Try a different port
python manage.py runserver 0.0.0.0:8080
6
